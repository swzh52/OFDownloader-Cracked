# OFDownloader Chrome 2.2.2 项目结构和功能分析

## 1. 项目概览

该目录是一个已经构建完成的 Chrome Manifest V3 扩展包，扩展名称为 `Onlyfans Downloader Chrome`，版本号为 `2.2.2`。从 `manifest.json`、入口脚本、资源目录和运行时代码可以判断，它的主要用途是在 OnlyFans 页面中注入下载按钮和下载侧边栏，解析图片、视频、聊天媒体和 PhotoSwipe 预览媒体，并通过扩展后台、扩展应用页、浏览器下载 API、OPFS 临时文件系统以及 WASM DRM 处理模块完成批量下载。

项目不是源码工程形态，而是发布产物形态：没有 `package.json`、源码目录、构建配置或未压缩的 Vue 组件源码。主要 JavaScript 文件已经被打包、压缩、混淆，变量名不可完全还原，但仍然能从 Manifest、路由命令、字符串、DOM 选择器、消息名和文件职责中分析整体结构。

## 3. Manifest 配置分析

`manifest.json` 是整个扩展的声明入口，核心信息如下：

| 配置项 | 内容 |
|---|---|
| Manifest 版本 | `3` |
| 扩展名称 | `Onlyfans Downloader Chrome` |
| 版本 | `2.2.2` |
| 后台脚本 | `background.js`，类型为 `module`，作为 MV3 Service Worker 运行 |
| 扩展应用页 | `app.html` |
| OnlyFans 内容脚本 | `content-scripts/content.js` |
| Web 登录桥接脚本 | `content-scripts/web-auth-bridge.js` |
| 公开可访问资源 | WASM、OPFS worker、`main-world.js`、图标、CSS、登录桥接脚本 |

扩展申请的权限包括：

- `cookies`：读取 OnlyFans 登录相关 cookie。
- `storage`：保存用户状态、设置、下载状态、鉴权快照等。
- `identity`：用于浏览器身份能力或 OAuth 相关流程。
- `scripting`：向页面注入脚本，例如把 `main-world.js` 注入到页面主世界。
- `activeTab`、`tabs`：查询、激活、打开、复用标签页和窗口。
- `downloads`、`downloads.open`：创建下载任务、打开或展示下载结果。

主机权限包括：

- `*://*.onlyfans.com/*`
- `*://r2.hlsdownloader.com/*`
- `*://hlsdownloader.com/*`
- `*://api.savemydayapp.com/*`

这说明扩展的业务边界主要在 OnlyFans 站点、官方/服务端站点 `hlsdownloader.com`、远程资源下载域和 `savemydayapp.com` API。

## 4. 核心文件职责

### 4.1 `background.js`

`background.js` 是扩展后台 Service Worker，也是全项目的中枢模块。它负责监听扩展消息、管理会话状态、处理浏览器 API 调用、打开应用窗口、处理订阅/登录跳转、代理下载请求，以及调度 DRM 相关逻辑。

主要功能包括：

- 读取 OnlyFans cookie，包括 `csrf`、`fp`、`sess`、`auth_id`、`st`、`ref_src` 等。
- 构造 OnlyFans 鉴权快照，包含 `userId`、`xbc`、`cookieHeader`、`userAgent`、更新时间等字段。
- 维护用户登录状态、会员状态和公开用户状态。
- 处理 `Session.GetSnapshot`、`Session.GetPublicUserStatus`、`Session.GetOnlyFansAuthState`、`Session.RefreshUserInfo`、`Session.RefreshSiteAuth`、`Session.SignOut` 等会话命令。
- 处理 Web 登录回调，支持 `hlsdownloader.com` 与本地开发源的认证桥接。
- 打开或复用 `app.html` 扩展窗口，用于主应用、下载页面、订阅管理页等。
- 代理浏览器下载 API，包括 `Downloads.Download`、`Downloads.Search`、`Downloads.Cancel`、`Downloads.Show`、`Downloads.Open`。
- 代理浏览器存储 API，包括 `Storage.Proxy.Get`、`Storage.Proxy.Set`、`Storage.Proxy.Remove`。
- 处理 `Drm.GetOnlyFansAuthState` 和 `Drm.RequestKey`，为 DRM 下载流程提供鉴权与密钥请求能力。
- 处理版本检查、更新提示、每日提示和站点授权提示。
- 处理导航命令，如打开首页、订阅页、订阅管理页、外部链接和扩展 App。
- 处理埋点上报，命令名中可见 `Analytics.Report`。

从代码结构看，后台中存在多个消息路由模块，例如：

| 路由名称 | 作用 |
|---|---|
| `web-auth` | 扩展登录、外部站点认证回调、窗口导航 |
| `downloads-proxy` | 代理浏览器下载 API |
| `browser-storage` | 代理浏览器存储读写 |
| `user-session` | 用户状态、OnlyFans 鉴权状态 |
| `site-auth-candidate` | 接收 OnlyFans 页面 `/users/me` 登录状态候选信息 |
| `download-session` | 创建、认领、更新下载窗口会话 |
| `navigation` | 打开 App、官网、订阅、外部链接 |
| `version-gate` | 版本检查、更新提示、更新入口 |
| `analytics` | 事件上报 |
| `drm-auth-snapshot` | 读取 DRM 所需 OnlyFans 鉴权状态 |
| `drm-key` | 请求 DRM key |
| `proxy-fetch` | 后台代理 fetch |

### 4.2 `content-scripts/content.js`

`content-scripts/content.js` 是注入到 OnlyFans 页面的隔离世界脚本。它运行在 `document_start`，负责初始化扩展在页面上的 UI 外壳，并和后台通信。

从可见字符串和组件结构看，它主要负责：

- 注入站内伪侧边栏 `ofdl-in-site-sidebar`。
- 注入每日提示/升级提示容器 `ofdl-daily-prompt`。
- 显示下载队列。
- 显示下载状态分类：`Processing`、`Completed`、`Failed`。
- 显示站内登录/会员状态，包括 `FREE`、`PRO`、`MAX`。
- 支持打开订阅管理入口。
- 支持输出目录选择。
- 支持 `Super Large Video Mode (SLVM)`，即大视频模式。
- 处理下载任务取消、清空、预览、自动折叠等交互。
- 通过消息端口订阅后台的 `Session.Updated` 会话状态更新。

该文件中包含大量 Vue 运行时代码和组件代码，说明站内侧边栏是一个 Vue 应用或 Vue 子应用打包后的产物。

### 4.3 `main-world.js`

`main-world.js` 是被注入到 OnlyFans 页面主世界的脚本。它的职责更接近“页面数据解析器”和“下载按钮注入器”，因为它能直接访问页面上的 Vue 实例、页面全局对象、`fetch`、`XMLHttpRequest` 和 OnlyFans 前端组件数据。

主要功能包括：

- Hook 页面 `fetch` 和 `XMLHttpRequest`。
- 监听 `/api2/v2/users/me` 请求结果，判断 OnlyFans 是否已登录。
- 将登录状态候选信息通过自定义事件发送给内容脚本/后台。
- 在动态页面中观察 DOM 变化。
- 给 Feed 帖子插入下载按钮。
- 给聊天消息媒体插入下载按钮。
- 给 PhotoSwipe 预览层插入下载按钮。
- 从 Vue props、DOM、PhotoSwipe 数据源中提取媒体任务。
- 解析图片、视频、GIF、DRM 视频等媒体类型。
- 提取媒体 URL、缩略图 URL、发布时间、媒体 ID、创作者用户名、正文文本等。
- 解析视频清晰度选项，例如原始源、转码源、DRM bundle、DRM 转码源。
- 显示清晰度下拉选择器。
- 对免费用户做能力限制，例如高分辨率视频、DRM 下载弹出升级提示。
- 通过页面事件 `invokeTasks` 把任务交给扩展侧处理。

可见的 DOM 选择器包括：

| 选择器 | 用途 |
|---|---|
| `.b-post__wrapper` | Feed 帖子容器 |
| `.b-chat__message__content` | 聊天消息容器 |
| `.pswp` | PhotoSwipe 图片/视频预览层 |
| `[data-ofdl-role="feed-parse-btn"]` | Feed 下载按钮 |
| `[data-ofdl-role="chat-parse-btn"]` | 聊天下载按钮 |
| `[data-ofdl-role="pswp-parse-btn"]` | 预览层下载按钮 |

### 4.7 `content-scripts/web-auth-bridge.js`

该脚本运行在：

- `http://localhost/*`
- `https://hlsdownloader.com/*`
- `https://www.hlsdownloader.com/*`

它的作用是连接网页登录页和扩展后台。网页侧发送认证回调消息后，该脚本把消息转发给扩展后台的 `web-auth` 路由，再把后台处理结果回传给网页。

它只接受特定来源和特定消息结构，例如：

- 来源：`hlsdownloader-website`
- 类型：`AUTH_CALLBACK`
- 通道：`ofdl:web-auth-page-bridge:v1`

### 4.8 `web-auth-page-bridge.js`

该文件是注入到网页主世界的桥接脚本。它重写或包装网页可见的 `chrome.runtime.sendMessage`，让 `hlsdownloader.com` 页面可以像调用扩展消息一样提交认证回调。

它和 `content-scripts/web-auth-bridge.js` 配合形成两段式桥接：

```text
hlsdownloader.com 页面
    ↓ chrome.runtime.sendMessage 包装
web-auth-page-bridge.js
    ↓ window.postMessage
content-scripts/web-auth-bridge.js
    ↓ runtime.sendMessage
background.js
```

### 4.9 `assets/opfs-worker-TYoFt98x.js`

这是一个 OPFS 文件写入 Worker。OPFS 是 Origin Private File System，用于在浏览器内部存储临时大文件。

它支持的消息类型包括：

| 消息类型 | 作用 |
|---|---|
| `open` | 打开或创建临时文件 |
| `write` | 按位置写入二进制数据 |
| `getSize` | 获取当前文件大小 |
| `read` | 从指定偏移读取二进制块 |
| `close` | flush 并关闭写入句柄 |
| `remove` | 删除临时文件 |

代码优先使用 `createSyncAccessHandle()`，如果不可用则退回 `createWritable()`。这说明它被用于大文件下载、分片写入、合并或临时缓存。

### 4.10 `assets/wasm/ofdl_core.js` 与 `.wasm`

`ofdl_core.js` 是 WASM 绑定层，`ofdl_core_bg.7ee0e04c0fad.wasm` 是实际 WASM 二进制。导出的核心类和函数包括：

- `WasmDrmCoreSession`
- `wasm_core_capabilities()`
- `wasm_core_version()`

`WasmDrmCoreSession` 可见的方法包括：

| 方法 | 推测作用 |
|---|---|
| `set_key` | 设置 DRM 解密 key |
| `push_encrypted_chunk` | 输入加密媒体分片 |
| `pull_output_chunk` | 拉取解密或处理后的输出分片 |
| `finalize_track` | 完成单轨道处理 |
| `remux_finalize` | 完成封装/重封装 |
| `abort` | 中止任务 |
| `close` | 关闭会话 |

结合后台中的 `Drm.RequestKey`、OnlyFans 鉴权快照和媒体解析逻辑，可以判断该模块用于 DRM 视频下载时的解密、分片处理或重封装。

### 4.11 `assets/app-BzL_7htP.css` 与 `content-scripts/content.css`

`assets/app-BzL_7htP.css` 是扩展 App 的主样式文件。

`content-scripts/content.css` 是 OnlyFans 页面内注入 UI 的样式文件，主要服务于站内侧边栏、提示、按钮、下载队列和覆盖层 UI。

### 4.12 `icon/`

图标目录提供矩形和圆角矩形两套图标，尺寸包括 16、48、128。Manifest 默认使用 `icon-rect-*`，部分站内 UI 或 web accessible resources 中也暴露了 48px 图标。

## 5. 功能流程分析

### 5.1 扩展启动流程

1. Chrome 加载 `manifest.json`。
2. MV3 Service Worker 启动 `background.js`。
3. 当用户访问 OnlyFans 页面时，浏览器在 `document_start` 注入 `content-scripts/content.js`。
4. 内容脚本初始化站内 UI，并注入或加载 `main-world.js`。
5. `main-world.js` 进入页面主世界，开始监听 DOM、Hook 网络请求并解析媒体。
6. 用户点击扩展按钮时，后台打开或复用 `app.html` 扩展应用窗口。

### 5.2 OnlyFans 登录状态检测流程

1. `main-world.js` Hook 页面 `fetch` 和 `XMLHttpRequest`。
2. 当页面请求 `/api2/v2/users/me` 时，脚本读取响应中的登录状态。
3. 登录状态通过自定义事件发送出来。
4. 内容脚本或后台接收后触发 `SiteAuth.CandidateReady`。
5. 后台进一步读取浏览器 cookie，确认 `auth_id`、`fp`、cookie header 等是否存在。
6. 后台生成或刷新 OnlyFans 鉴权快照。
7. 扩展 App 和站内侧边栏同步显示授权状态。

### 5.3 媒体解析与下载任务生成流程

1. `main-world.js` 用 `MutationObserver` 观察 OnlyFans 页面动态变化。
2. 当发现 Feed、聊天消息或 PhotoSwipe 预览层时，插入下载按钮。
3. 用户点击下载按钮。
4. 脚本从 Vue props、页面 DOM 或 PhotoSwipe 数据源中提取媒体信息。
5. 对媒体进行分类：
   - 图片
   - 视频
   - GIF
   - DRM 视频
6. 为视频构造清晰度列表，包括原始源、转码源、DRM 源等。
7. 如果用户选择了清晰度，则按可用清晰度更新任务 URL。
8. 若免费用户尝试下载高分辨率或 DRM 内容，则弹出升级提示。
9. 下载任务通过页面事件和扩展消息进入下载队列。

### 5.4 下载处理流程

1. 下载任务进入站内侧边栏或扩展 App 的任务队列。
2. 后台通过下载代理、fetch 代理或 OPFS worker 处理文件写入。
3. 普通文件可能直接走浏览器 `downloads.download`。
4. 大文件或需要合并的任务可能先写入 OPFS 临时文件。
5. DRM 内容会经过后台鉴权、key 请求和 WASM 核心处理。
6. 下载状态进入：
   - `Processing`
   - `Completed`
   - `Failed`
7. 完成的任务可通过浏览器下载 API 打开或显示。

### 5.5 订阅与会员流程

扩展存在免费、PRO、MAX 等状态。页面中可见以下功能：

- 免费用户下载数量或质量受限。
- PRO / MAX 用户解锁更多下载能力。
- MAX 页面强调多创作者批量下载和更多媒体源。
- 订阅升级可能通过 Stripe。
- 取消订阅页面包含原因选择、反馈输入、Stripe Customer Portal 打开和资料同步。
- 站内侧边栏可以显示当前会员状态，并打开订阅管理页。

### 5.6 Web 登录桥接流程

1. 用户从扩展内打开 `hlsdownloader.com/login` 或相关登录页。
2. 网页侧产生 `AUTH_CALLBACK` 消息。
3. `web-auth-page-bridge.js` 包装网页 `chrome.runtime.sendMessage`。
4. 消息通过 `window.postMessage` 进入 `content-scripts/web-auth-bridge.js`。
5. 内容脚本调用 `runtime.sendMessage` 发送给后台。
6. 后台保存用户凭证和会员状态。
7. 扩展 App 或 OnlyFans 站内 UI 收到会话更新。

## 6. 主要数据与状态

从共享运行时代码和后台逻辑可见，扩展维护以下类型的数据：

| 状态 | 用途 |
|---|---|
| 用户状态 | 邮箱、token、会员类型、会员时间、订阅状态 |
| OnlyFans 鉴权快照 | `auth_id`、`fp/xbc`、cookie header、UA |
| Web 登录下一跳 | 登录完成后返回的扩展内部路由 |
| 创作者集合缓存 | 批量下载或创作者列表缓存 |
| 版本检查状态 | 当前版本、最新版本、更新提示状态 |
| 设置项 | 下载方式、输出目录、侧边栏行为等 |
| 敏感缓存设备 key | 加密或敏感缓存相关 |
| 下载队列与下载会话 | 待处理、成功、失败任务 |

## 7. 权限与安全关注点

该扩展需要较高权限才能实现功能，尤其是：

- 读取 OnlyFans cookie。
- 访问 OnlyFans 页面内部数据。
- 注入页面主世界脚本。
- 代理下载和网络请求。
- 处理 DRM key 与媒体分片。
- 与外部站点交换登录回调。

这些权限从功能角度是合理的，但也意味着使用者需要信任该扩展及其服务端。尤其需要关注：

- `cookies` 权限会读取登录相关 cookie。
- `host_permissions` 覆盖 OnlyFans 和扩展服务端。
- 用户 token 和 OnlyFans 鉴权快照会进入扩展存储。
- DRM key 请求依赖后台和远端服务。
- 外部认证桥接需要严格限制来源，当前代码中确实只允许 `hlsdownloader.com`、`www.hlsdownloader.com` 和本地开发源。

## 8. 技术栈判断

## 9. 总体功能总结

该项目是一个面向 OnlyFans 的批量下载 Chrome 扩展发布包。它通过内容脚本和主世界注入脚本在 OnlyFans 页面中增加下载能力，能够解析 Feed、聊天和图片预览层中的媒体资源，生成图片、视频、GIF、DRM 视频等下载任务，并提供清晰度选择、下载队列、任务状态、输出目录、大文件模式和下载结果管理。

后台 Service Worker 负责所有跨域、权限和浏览器能力相关操作，包括 cookie 读取、OnlyFans 授权确认、用户会话、订阅管理、下载代理、存储代理、窗口管理、版本检查、DRM key 请求和消息路由。扩展 App 则提供完整的独立管理界面，包括批量下载、下载历史、设置、订阅、升级、取消订阅和邀请等页面。

从工程形态看，这不是可直接二次开发的源码目录，而是已经编译后的扩展包。若要深入修改业务逻辑，需要反向分析压缩后的 JS，或找到未打包源码工程；若只是安装、审计、理解结构和功能，当前目录已经包含完整运行所需资源。
